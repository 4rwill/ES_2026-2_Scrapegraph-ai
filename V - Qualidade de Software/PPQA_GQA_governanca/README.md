# PPQA/GQA - Governanca

## Escopo

Este diretorio consolida as evidencias de governanca de processo e produto usadas na auditoria do Eixo 5 (Qualidade de Software), com alinhamento a CMMI-PPQA e MPS.BR-GQA.

## Arquivos analisados e foco analitico

| Arquivo | O que foi analisado |
|---|---|
| `CONTRIBUTING.md` | Regras formais de contribuicao (fluxo de branch, padroes de commit, expectativas de testes e padrao de codigo). |
| `.releaserc.yml` | Politica de versionamento e rastreabilidade por Conventional Commits, incluindo regras de classificacao de mudancas. |
| `SEMANTIC_COMMITS.md` | Grau de formalizacao do uso de commits semanticos e evidencia de governanca documental para mensagens de commit. |
| `SECURITY.md` | Procedimento institucional para reporte de vulnerabilidades e canal oficial de comunicacao para incidentes de seguranca. |
| `PullRequests/PR_1027_reviews.md` | Evidencia de revisao tecnica em PR (peer review), incluindo criterio de aceitacao, justificativa tecnica e registro de parecer. |

## Sintese tecnica

A analise desta pasta foi utilizada para avaliar se as regras de qualidade estao explicitadas, comunicadas e rastreaveis no processo de desenvolvimento.

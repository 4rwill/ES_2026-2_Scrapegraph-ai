# PPQA/GQA - Nao Conformidades

## Escopo

Este diretorio reune os artefatos usados para examinar como o projeto identifica, registra e trata nao conformidades tecnicas e de seguranca.

## Arquivos analisados e foco analitico

| Arquivo | O que foi analisado |
|---|---|
| `.github/ISSUE_TEMPLATE/bug_report.md` | Estrutura formal para registro de defeitos, nivel de detalhamento exigido e potencial de rastreabilidade. |
| `.github/ISSUE_TEMPLATE/feature_request.md` | Mecanismo de captura estruturada de mudancas, com foco em clareza de necessidade e criterios de evolucao. |
| `.github/ISSUE_TEMPLATE/custom.md` | Grau de padronizacao para registros livres e risco de variacao na qualidade dos relatos. |
| `PullRequests/PR_1027_reviews.md` | Evidencia concreta de tratamento de nao conformidade em revisao de PR e justificativa tecnica de decisao. |
| `SECURITY.md` | Processo formal de comunicacao de vulnerabilidades, considerado como fluxo de nao conformidade de seguranca. |

## Sintese tecnica

A analise desta pasta foi utilizada para avaliar se existe trilha auditavel para abertura, avaliacao e encaminhamento de problemas ao longo do ciclo de desenvolvimento.
